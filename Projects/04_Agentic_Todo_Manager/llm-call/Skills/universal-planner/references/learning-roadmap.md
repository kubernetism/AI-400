# Learning Roadmap Reference

Use this reference when the plan is primarily about learning a skill, technology, or body of knowledge.

---

## Learning Plan Structure

### Step 1: Skill Audit
- What do you already know? (Baseline)
- What is the target proficiency? (Beginner → Intermediate → Advanced → Expert)
- How much time per day/week can you invest?

### Step 2: Decompose the Domain
Break the target skill into:
1. **Foundation concepts** (must know before anything else)
2. **Core skills** (the 80% you'll use day-to-day)
3. **Advanced topics** (10% you need for complex situations)
4. **Specializations** (niche areas to explore after mastery)

### Step 3: Resource Mapping
For each topic, identify:
- Primary resource (book, course, official docs)
- Practice vehicle (exercises, projects, labs)
- Validation method (quiz, project, certification)

### Step 4: Spaced Repetition Schedule
- Review new material: same day, 3 days, 1 week, 2 weeks, 1 month
- Build this into the plan as "Review Sessions"

## Time Estimates by Proficiency Level

| Domain | Beginner → Functional | Functional → Advanced |
|---|---|---|
| Programming Language | 40–80 hours | 200–500 hours |
| DevOps Tool (Kubernetes, Docker) | 30–60 hours | 100–200 hours |
| ML/AI Framework | 50–100 hours | 300+ hours |
| Cloud Platform | 40–80 hours | 150–300 hours |
| Framework (FastAPI, React) | 20–40 hours | 80–150 hours |
| Certification (CKA, AWS) | 60–120 hours prep | N/A |

---

## Project-Based Learning Milestones

Never let learning be purely theoretical. After each milestone, build something:

| Topic | Practice Project |
|---|---|
| Kubernetes Basics | Deploy a 2-service app with Services + Ingress |
| Kubernetes Security | Implement RBAC + NetworkPolicy on local cluster |
| LangChain | Build a RAG chatbot over custom documents |
| FastAPI | Create a CRUD API with auth and Docker deployment |
| LangGraph | Build a multi-agent research assistant |

## Study Session Template (90-min block)

```
00:00 – 00:10  Review previous session notes (warm-up)
00:10 – 00:50  New content (read/watch/code)
00:50 – 01:10  Practice / apply (lab, exercise, mini-project)
01:10 – 01:20  Summarize in own words (spaced repetition)
01:20 – 01:30  Note questions, plan next session
```
